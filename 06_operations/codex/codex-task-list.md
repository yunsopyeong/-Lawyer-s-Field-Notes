# 윤소평 변호사 법률 AI 에이전트 프로젝트  
## 코덱스로 맡길 업무 20개 실무표

이 문서는 윤소평 변호사 법률 AI 에이전트 프로젝트에서  
코덱스(Codex)에 맡기기 적합한 업무를 정리한 운영 문서이다.

목적은 단순하다.

- 채팅으로 하는 것이 더 나은 일
- 코덱스로 넘기는 것이 더 효율적인 일

을 구분하고,  
특히 GitHub 저장소 기반으로 반복 가능한 구조를 만드는 데 있다.

---

## 1. 기본 원칙

코덱스는 주로 아래와 같은 일에 투입한다.

- 여러 파일을 한 번에 만들거나 수정하는 일
- 저장소 구조를 정리하는 일
- 반복 문서를 자동 생성하는 일
- 검수 기준을 코드나 규칙으로 옮기는 일
- 브랜치 단위 작업과 PR 생성이 잘 맞는 일
- 사람이 매번 손으로 하면 시간이 많이 드는 일

반대로 아래 업무는 코덱스보다 채팅형 작업이 더 낫다.

- 법률 설명 자체
- 블로그 글 1편 작성
- 유튜브 대본 1편 작성
- 판례 취지 요약
- 상담형 문장 수정
- FAQ 문장 생성

즉,  
콘텐츠 작성은 채팅 중심,  
콘텐츠 생산 시스템 구축은 코덱스 중심으로 운영한다.

## 3. 우선순위 추천

### 최우선
아래 5개는 가장 먼저 코덱스로 돌려볼 만한 업무다.

1. 저장소 기본 폴더 구조 생성
2. README 자동 정리
3. 이슈 템플릿 생성
4. PR 템플릿 생성
5. 분야별 대표자료 파일 분리

이 5개가 먼저 정리되면  
나머지 자동화 작업도 훨씬 안정적으로 붙일 수 있다.

---

## 4. 코덱스 투입 판단 기준

아래 질문에 3개 이상 “예”가 나오면 코덱스로 넘기는 것이 좋다.

- 이 작업은 파일이 여러 개 바뀌는가
- 반복해서 다시 하게 될 가능성이 큰가
- 폴더 구조와 파일명 규칙이 중요한가
- 사람이 매번 손으로 하면 번거로운가
- 나중에 GitHub에서 diff 비교가 필요할까
- 검수 기준을 일정하게 유지해야 하는가
- 문서 자동 생성 또는 일괄 수정이 필요한가

3개 이상 해당하면  
채팅보다 코덱스가 더 적합할 가능성이 크다.

## 5. 채팅과 코덱스의 역할 구분

### 채팅이 더 적합한 업무
- 법률 쟁점 설명
- 판례 해설
- 블로그 원고 작성
- 브런치 글 작성
- 유튜브 대본 작성
- FAQ 문장 다듬기
- 상담형 말투 수정

### 코덱스가 더 적합한 업무
- 템플릿 파일 생성
- 저장소 구조 정리
- 다수 문서 일괄 수정
- 검수 도구 작성
- 자동화 스크립트 작성
- 중복 파일 정리
- 브랜치 단위 작업 분리
- PR 생성 전 변경 정리

## 6. 실무 운영 예시

예를 들어 “개인회생/개인파산 주제 파일을 정리하고 싶다”면 흐름은 아래처럼 잡는다.

### 예시 작업 흐름
1. 이슈 생성  
   - `[소스] 개인회생/개인파산 대표자료 구조화`

2. 브랜치 생성  
   - `draft/personal-rehab-bankruptcy-structure`

3. 코덱스에 맡길 일  
   - 기존 통합 자료를 파일별로 분리
   - 폴더 구조 생성
   - README 또는 index 파일 연결
   - 파일명 통일
   - 링크 누락 점검

4. PR 생성  
   - `[소스] 개인회생/개인파산 대표자료 파일 구조 정리`

이렇게 하면  
한 번의 채팅 결과가 아니라  
반복 가능한 저장소 자산으로 남는다.

## 7. 최종 원칙

코덱스는 “법률 판단을 대신하는 도구”가 아니다.  
코덱스는 “법률 콘텐츠 생산 시스템을 만드는 도구”에 가깝다.

따라서 이 프로젝트에서는 아래처럼 이해하면 된다.

- 법률 설명과 문장 작성은 채팅 중심
- 구조화, 자동화, 리팩터링, 검수 시스템은 코덱스 중심

즉,  
콘텐츠는 사람이 판단하고,  
구조와 반복은 코덱스로 정리한다.

---

## 8. 한 줄 정리

윤소평 변호사 법률 AI 에이전트 프로젝트에서 코덱스는  
“글을 대신 쓰는 도구”가 아니라  
“글이 쌓이고, 정리되고, 자동화되도록 만드는 운영 도구”로 쓰는 것이 가장 효율적이다.
