# AGENTS.md

## 목적

이 저장소는 윤소평 변호사 법률 AI 에이전트 프로젝트의 기준 문서, 프롬프트, 소스 자료, 발행형 원고를 관리한다.  
Codex 또는 다른 에이전트는 아래 기준을 우선 적용한다.

---

## 1. 최우선 작업 원칙

- 답변과 문서는 항상 **정보 수집 → 분석 → 결론** 구조를 기본으로 한다.
- 대한민국 법령, 판례, 공식자료를 우선 반영한다.
- 회생·파산 분야는 서울회생법원 실무준칙과 실제 자료 준비 포인트를 함께 본다.
- 과장광고, 결과 보장, 공포 조장 표현을 사용하지 않는다.
- 질문과 직접 관련 있는 경우에만 참고자료 링크를 연결한다.
- 기존 문서를 중복 생성하기보다, 먼저 기존 파일을 보강할 수 있는지 검토한다.

---

## 2. 문체 원칙

- 변호사가 의뢰인에게 차분히 설명하는 말투
- 짧고 분명한 문장
- 법률 용어 뒤에는 쉬운 설명을 덧붙임
- 실무상 오해 포인트를 반드시 포함
- 결론은 광고형이 아니라 체크포인트 정리형

---

## 3. 작업 전 반드시 확인할 것

1. 이 작업이 기존 파일 수정인지, 새 파일 생성인지
2. 저장 위치가 맞는지
3. 공개 저장소에 올려도 되는 문서인지
4. 민감정보, 사건정보, 비공개 자료가 포함되지 않는지
5. README, AGENTS, 운영 문서와 충돌하지 않는지

---

## 4. 저장소 작업 규칙

- `main` 브랜치에는 직접 커밋하지 않는다.
- 작업은 `draft/*` 또는 `hotfix/*` 브랜치에서 진행한다.
- 한 PR에는 한 주제만 담는다.
- README, 프롬프트, 템플릿, 소스 자료를 한 PR에 과하게 섞지 않는다.
- 같은 주제의 중복 파일을 새로 만들지 않는다.

---

## 8. Codex 사용 기준

Codex는 아래 작업에 우선 사용한다.

- 여러 파일을 한 번에 만들거나 수정하는 일
- 저장소 구조를 정리하는 일
- 반복 문서를 자동 생성하는 일
- 검수 기준을 코드나 규칙으로 옮기는 일
- 브랜치 단위 작업과 PR 생성이 잘 맞는 일

반대로 아래 작업은 일반 채팅 또는 수동 작성이 더 적합하다.

- 법률 설명 자체
- 블로그 글 1편 작성
- 유튜브 대본 1편 작성
- 판례 취지 요약
- 상담형 문장 수정

---

## 9. 최종 점검

작업 후 아래를 확인한다.

- 구조가 정보 수집 → 분석 → 결론을 지키는가
- 과장 표현이 없는가
- 민감정보가 없는가
- 링크가 끊기지 않는가
- 기존 문서와 충돌하지 않는가
