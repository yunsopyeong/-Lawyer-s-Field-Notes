# 윤소평 변호사 법률 콘텐츠 시스템

이 저장소는 윤소평 변호사 프로젝트를 기준으로  
대한민국 법률정보, 법률 콘텐츠, 프롬프트, 소스 자료, 운영 문서를 체계적으로 관리하기 위한 작업 저장소입니다.

회생·파산을 중심으로 하되,  
민사, 형사, 가사, 상속, 성년후견 등 관련 생활법률 콘텐츠까지 함께 다룹니다.

---

## 저장소 목적

이 저장소의 목적은 아래와 같습니다.

- 법률 콘텐츠 초안과 최종본을 분리해 관리
- 네이버 블로그, 브런치, 네이버 프리미엄콘텐츠, 유튜브 대본을 주제별로 축적
- 커스텀 GPT용 프롬프트와 내부 운영 기준을 정리
- 법령, 판례, 공식자료, 실무준칙, 내부 요약본을 소스 형태로 관리
- 반복 가능한 법률 콘텐츠 생산 구조를 만드는 것

---

## 핵심 원칙

이 저장소의 모든 문서와 결과물은 아래 원칙을 기본으로 합니다.

- 대한민국 법령, 판례, 공식자료를 우선 반영합니다.
- 회생·파산 분야는 서울회생법원 실무준칙도 함께 고려합니다.
- 답변과 원고는 정보 수집 → 분석 → 결론 구조를 기본으로 합니다.
- 과장광고, 단정 표현, 공포 조장 문구는 사용하지 않습니다.
- 질문과 직접 관련 있을 때만 참고자료 링크를 연결합니다.
- 원고는 실제 상담을 해 본 변호사의 문장처럼 차분하고 쉽게 설명합니다.

## AGENTS.md 운영 원칙

이 저장소는 Codex와 함께 작업할 수 있도록 프로젝트 전용 지침 파일인 `AGENTS.md`를 사용합니다.

`AGENTS.md`는 이 저장소에서 문서를 만들거나 수정할 때 따라야 하는 기본 기준을 정리한 파일입니다.  
쉽게 말하면, 사람에게는 운영 매뉴얼이고 Codex에게는 프로젝트별 작업 기준서에 가깝습니다.

이 파일에는 주로 아래 내용이 들어갑니다.

- 어떤 순서로 작업할지
- 어떤 파일을 먼저 읽어야 하는지
- 법률정보를 어떤 우선순위로 정리할지
- 어떤 문체를 사용할지
- 어떤 표현을 피해야 하는지
- 수정 후 무엇을 점검해야 하는지
- Git 브랜치와 변경 보고를 어떻게 할지

윤소평 변호사 프로젝트에서는 `AGENTS.md`를 통해 아래 방향을 공통 기준으로 유지합니다.

- 답변과 문서는 정보 수집 → 분석 → 결론 구조를 기본으로 합니다.
- 대한민국 법률, 판례, 공식자료, 실무준칙을 우선 반영합니다.
- 회생·파산 분야는 대표이사, 사업주, 채무자 관점의 실무 설명을 강화합니다.
- 블로그, 브런치, 프리미엄콘텐츠, 유튜브 대본은 매체별 문체 차이를 둡니다.
- 과장광고, 결과 보장, 공포 조장 표현은 사용하지 않습니다.
- 관련 없는 자료를 억지로 추천하지 않습니다.

프로젝트 전체에 적용되는 공통 규칙은 저장소 루트의 `AGENTS.md`에서 관리합니다.  
필요하면 특정 폴더 안에 별도 `AGENTS.md`를 두어 하위 작업 규칙을 더 구체화할 수 있습니다.

## 브랜치 운영 규칙

이 저장소는 `main` 브랜치를 최종본 보관용으로 사용합니다.  
초안, 수정, 실험 작업은 반드시 별도 브랜치에서 진행합니다.

### 기본 원칙
- `main`은 최종 승인된 문서만 반영합니다.
- 직접 `main`에서 작업하지 않습니다.
- 모든 수정은 작업 브랜치에서 진행한 뒤 Pull Request로 병합합니다.
- 작업이 끝난 브랜치는 병합 후 삭제합니다.

### 브랜치 이름 규칙
- `draft/blog-*` : 네이버 블로그 원고 작업
- `draft/youtube-*` : 유튜브 대본 작업
- `draft/prompt-*` : 프롬프트 수정
- `draft/source-*` : 소스 자료 정리
- `hotfix/*` : 긴급 수정

### 브랜치 이름 예시
- `draft/blog-corporate-bankruptcy-overview`
- `draft/blog-personal-rehabilitation-eligibility`
- `draft/youtube-rehabilitation-vs-bankruptcy`
- `draft/prompt-blog-main-v2`
- `draft/source-seoul-court-guidelines-update`
- `hotfix/readme-links-fix`

## 이슈 템플릿 운영 방식

이 저장소는 작업 유형별로 이슈를 나눠 관리합니다.

- 블로그 원고 요청
- 유튜브 대본 요청
- 소스 자료 업데이트
- 긴급 수정 요청

이슈를 먼저 만들고,  
그 이슈에 맞는 브랜치를 따로 생성한 뒤 작업을 진행하는 방식을 권장합니다.

예시:
- 이슈: `[블로그] 법인파산 절차 글 작성`
- 브랜치: `draft/blog-corporate-bankruptcy-procedure`
- PR: `[블로그] 법인파산 절차 글 초안 추가`

---

## Pull Request 운영 원칙

PR은 단순 병합 요청이 아니라  
최종 반영 전에 문서 품질을 점검하는 단계입니다.

PR에서는 아래를 확인합니다.

- 정보 수집 → 분석 → 결론 구조 유지 여부
- 법률 근거 반영 여부
- 실무상 오해 포인트 반영 여부
- 과장 표현 제거 여부
- 제목, 소제목, FAQ, CTA의 자연스러움
- 관련 없는 링크를 억지로 넣지 않았는지 여부

- 당장 상담받지 않으면 큰일 납니다

## 이 저장소를 이렇게 쓰면 좋습니다

이 저장소는 단순 파일 보관함이 아니라  
윤소평 변호사 프로젝트의 작업실처럼 쓰는 것이 좋습니다.

즉,

- `main`은 최종본 보관소
- `draft` 브랜치는 초안 작업실
- 이슈는 작업 지시서
- PR은 검수 및 반영 요청서
- `AGENTS.md`는 프로젝트 공통 작업 기준서

이 구조로 운영하면  
법률 콘텐츠, 프롬프트, 소스 문서를 주제별로 분리하면서도  
나중에 다시 찾기 쉬운 형태로 축적할 수 있습니다.
