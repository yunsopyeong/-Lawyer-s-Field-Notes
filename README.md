# 윤소평 변호사 프로젝트 GitHub 업로드 5개 파일 세트

이 저장소 세트는 프로젝트 대화에서 반복적으로 나온 운영 원칙, 프롬프트, 템플릿, 주제 구조를 GitHub에 올리기 쉽게 5개 파일로 정리한 업로드본입니다.

## 목적
- 프로젝트 대화를 그대로 쌓는 대신 재사용 가능한 문서 구조로 정리
- 네이버 블로그, 브런치, 네이버 프리미엄콘텐츠, 유튜브 대본 작업의 기준 통일
- 회생·파산 중심 법률 콘텐츠 시스템을 GitHub에서 버전관리

## 권장 폴더 구조
- `README.md`
- `01_brand/operating-identity.md`
- `02_prompts/content-generation-prompts.md`
- `03_templates/info-analysis-conclusion-template.md`
- `04_topics/rehabilitation-bankruptcy/project-topic-map.md`

## 이 5개 파일의 역할
### 1. README.md
저장소의 용도, 구조, 사용 순서를 설명하는 시작 파일입니다.

### 2. 01_brand/operating-identity.md
윤소평 변호사 프로젝트의 기본 정체성, 문체, 답변 원칙, 금지 표현의 방향을 담은 운영 기준입니다.

### 3. 02_prompts/content-generation-prompts.md
네이버 블로그, 브런치, 프리미엄콘텐츠, 유튜브 대본 작성에 바로 붙여 넣을 수 있는 핵심 프롬프트 모음입니다.

### 4. 03_templates/info-analysis-conclusion-template.md
모든 답변과 글의 공통 뼈대인 정보 수집 → 분석 → 결론 템플릿입니다.

### 5. 04_topics/rehabilitation-bankruptcy/project-topic-map.md
법인파산, 법인회생, 개인회생, 개인파산 등 핵심 분야를 주제별로 정리한 프로젝트 맵입니다.

## 사용 순서
1. README로 전체 구조 확인
2. operating-identity로 문체와 원칙 맞추기
3. content-generation-prompts에서 매체별 프롬프트 선택
4. info-analysis-conclusion-template으로 글 구조 잡기
5. project-topic-map에서 주제별 핵심 포인트 확인

## GitHub 업로드 방법
1. 저장소를 엽니다.
2. `Add file`을 누릅니다.
3. `Upload files`를 선택합니다.
4. 이 압축을 풀어서 폴더째 드래그합니다.
5. 커밋 메시지를 적고 저장합니다.

## 운영 팁
- 프로젝트 대화 전체를 원문으로 누적하지 말고, 재사용 가능한 문서만 업데이트합니다.
- 같은 주제가 반복되면 새 파일을 늘리기보다 기존 파일을 수정합니다.
- 법령 개정, 실무준칙 개정, 대표 영상 추가가 있으면 topic-map과 prompts부터 업데이트합니다.
